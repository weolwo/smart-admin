# QLExpress 3.x 升级 4.x 避坑与架构演进备忘录

**文档说明：** 本文档不仅总结了引擎语法的变迁，更包含了针对高并发、高可用生产环境的**架构级避坑指南**。所有新接手规则引擎的开发者，务必熟读！

## ⚙️ 一、 核心底座与内存模型 (Core & Memory)

| **升级维度** | **3.x 时代 (旧纪元)**            | **4.x 时代 (新纪元)**                | **⚠️ 架构师避坑与实战指南**                                   |
| ------------ | -------------------------------- | ------------------------------------ | ------------------------------------------------------------ |
| **解析引擎** | 易报错的手搓状态机               | **基于 Antlr4 的 AST 语法树**        | 语法容错率大幅提升。底层彻底拥抱函数式编程与 Lambda 。       |
| **底层传参** | 强依赖 `ThreadLocal` 隐式传递    | **绝对无状态，AST 方法入参递归流转** | **【红线】** 根除了 Tomcat 等容器下的内存泄漏隐患。业务侧严禁在底层复活 ThreadLocal，如需隐式传参，必须使用封装好的 `RuleContextHolder` 并严格在 `finally` 中 `clear`。 |
| **全局污染** | 脚本定义的变量会自动写入 Context | **默认绝对隔离，不污染宿主 Context** | `polluteUserContext` 默认为 `false`，脚本跑完即销毁，保证了 Java 外部宿主环境的纯洁性 。 |

## 🛡️ 二、 生产安全与运行暗坑 (Security & Runtime Traps)

| **升级维度** | **3.x 时代 (旧纪元)**      | **4.x 时代 (新纪元)**                        | **⚠️ 架构师避坑与实战指南**                                   |
| ------------ | -------------------------- | -------------------------------------------- | ------------------------------------------------------------ |
| **默认权限** | 黑名单“裸奔”，极易引发 RCE | **默认“绝对隔离”，禁调 Java 对象属性与方法** | **【红线】** 严禁开启 `SecurityStrategy.open()` ！所有系统能力必须通过 `@ScriptFunction` 包装的 SPI 接口（自定义函数）向脚本暴露。 |
| **缓存机制** | -                          | **支持 `cache(true)` 开启 AST 编译缓存**     | **【高危 OOM】** 缓存底层无大小限制 ！若是前端传入极其随机的动态脚本，极易撑爆 JVM！对于高频动态拼装脚本，务必定期调用 `clearCompileCache` 。 |
| **超时控制** | 统一强行中断               | **Java 回调中属于“假超时”**                  | **【高危卡死】** 引擎无法强行中断 Java 内部线程 。若通过 `@ScriptFunction` 调用的 HTTP/DB 操作卡死，脚本会无视超时配置一直阻塞。耗时 API 务必在 Java 内部做超时控制！ |

## ✍️ 三、 语法糖与前端编辑器 (Monaco IDE) 联调方案

| **升级维度** | **3.x 时代 (旧纪元)**         | **4.x 时代 (新纪元)**                     | **💡 架构师实战神技 (联动前端)**                              |
| ------------ | ----------------------------- | ----------------------------------------- | ------------------------------------------------------------ |
| **Map 声明** | 极其反人类的 `NewMap(a:1)`    | **天下大同的原生 JSON：`{"a": 1}`**       | **【神技：省库表】** 彻底废弃数据库 JSON 配置表！业务人员可直接在脚本中用 JSON 定义配置，前端 Vue3 Monaco 天然支持此语法的格式化与高亮！ |
| **语法校验** | 运行时才抛异常                | **提供 `check(script)` AST 预校验 API**   | **【神技：实时纠错】** 暴露后端校验接口给 Monaco Editor。运营人员敲击代码时，若有错误（如缺括号），立马捕获 `QLSyntaxException` ，在前端划红线提示！ |
| **换行规则** | 必须写分号 `;`                | **支持省略分号** **，但多行拼接极其严格** | **【避坑】** 如果多行拼接，**操作符必须留在上一行末尾** (如 `金额 + \n 补贴`) 。需在前端编辑器悬浮文档中加粗标红提示！ |
| **字符串**   | `'a'` 变 `char`，易报比对错误 | 全部解析为 `String` ；支持 `${exp}` 插值  | 彻底解决 `'a' == "a"` 为 false 的问题。拼接日志或提示语时，全面改用动态插值 `${}` 。 |
